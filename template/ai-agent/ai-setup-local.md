# AI設定生成器（リポジトリ用）

このファイルをAIエージェントに読み込ませると、リポジトリ用の `AGENTS.md` と `CLAUDE.md` を生成できる。

> 使い方：エージェントに
> 「`ai-setup-local.md` を読んで、このリポジトリの AGENTS.md と CLAUDE.md を生成して」と指示する。
> 既存ファイルがあれば差分更新でよい。

---

## 1. 生成するファイルと構成

```
<repo>/
├── AGENTS.md      ← マスター（実体）。リポジトリのルールはここに書く
├── CLAUDE.md      ← 中身は @AGENTS.md の1行のみ（import方式）
```

- マスターは **AGENTS.md** に一本化する（OpenAI / Google / GitHub Copilot が対応するオープン規格）。
- `CLAUDE.md` は import だけ。symlink はOS間の互換性が低いため使わない。
- Gemini CLI を使う場合は `~/.gemini/settings.json` の `contextFileName` を `AGENTS.md` に設定すれば別ファイル不要。

---

## 2. AGENTS.md に含める内容

エージェントは以下を生成する。`<...>` はユーザーに質問するか、リポジトリから推定して確認を取る。

### 2.1 プロジェクト概要

```markdown
# Project: <プロジェクト名>

## Stack

<言語・フレームワーク・DB・主要ツール>
```

### 2.2 作業ルール

プロジェクトに応じたルールを書く。以下は例。

```markdown
## 作業ルール

- <ビルド・テストの実行方法（例：`make test` 経由で実行、直接 `go test` しない）>
- <コード規約・リンターの指定があれば>
- <その他プロジェクト固有の注意事項>
```

リポジトリの構成を見て、必要なルールをユーザーに提案すること。
グローバル設定（応答言語・口調・コミット規約など）と重複する内容は書かない。

---

## 3. CLAUDE.md の中身（固定）

```markdown
@AGENTS.md
```

---

## 4. 生成手順（エージェントへの指示）

1. リポジトリの構成を調べ、Stack（2.1）を推定してユーザーに確認する。
2. リポジトリの Makefile・CI設定・リンター設定等から、作業ルール（2.2）を提案する。
3. `AGENTS.md` を生成する。
4. `CLAUDE.md` を生成する（中身は `@AGENTS.md` のみ）。
5. 既存ファイルがある場合は上書きせず差分提案する。
