# plan スキル

ユーザーの要件をもとに `.agents/plan/` に計画ファイルを作成する。

## 手順

1. ユーザーの要件を整理し、以下の構成で plan ファイルを作成する。
2. ファイル名はケバブケースの短い名前（例: `foo-bar.md`）。

## ファイル構成

```markdown
# タイトル

## 目的

何を・なぜやるのか。1〜3文。

## スコープ

- やること（箇条書き）

## 変更内容

具体的な変更の設計。必要に応じてコード例やディレクトリ構成を含める。

## 前提

- 着手に必要な条件や依存
```

## 注意

- 「スコープ」と「変更内容」はタスクの規模に応じて統合してよい。
- 不確定事項があればユーザーに確認してから書く。
